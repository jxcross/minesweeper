# 사용자 시나리오 (User Scenarios)

## 시나리오 1: 초보 플레이어의 초급 게임 완주 및 점수 등록

**페르소나**: 지뢰찾기를 처음 접하는 30대 사무직 사용자

**사전 조건**: 브라우저에서 `http://localhost:5173` 접속

### 흐름

1. 사용자가 앱을 열면 초급(9×9, 지뢰 10개) 보드가 `idle` 상태로 표시된다.
2. 상단 지뢰 카운터는 `010`, 타이머는 `000`, 스마일은 😀이다.
3. 사용자가 중앙 근처 셀(4,4)을 좌클릭한다.
   - 첫 클릭이므로 (4,4)와 인접 8칸을 제외한 영역에 지뢰 10개가 배치된다.
   - 타이머가 시작된다 (`idle → playing`).
   - BFS로 연결된 빈 칸들이 연쇄 공개된다.
4. 사용자가 숫자 힌트를 보며 지뢰 위치를 추론한다.
5. 지뢰로 의심되는 칸을 우클릭해 🚩 플래그를 세운다. 지뢰 카운터가 감소한다.
6. 모든 안전 칸을 공개하면 승리 조건이 충족된다.
   - Confetti 애니메이션이 3초간 재생된다.
   - 스마일이 😎으로 변경된다.
   - 타이머가 `087`에서 정지된다.
7. "점수를 등록하시겠습니까?" 모달이 표시된다.
8. 사용자가 이름 "Alice"를 입력하고 Submit을 클릭한다.
   - `POST /api/scores` 요청: `{ playerName: "Alice", difficulty: "beginner", timeSeconds: 87 }`
   - 서버가 `201 Created`로 응답한다.
9. 리더보드 모달이 열리며 Alice의 기록이 1위에 표시된다.

**기대 결과**: 게임 완주 + 점수 저장 + 리더보드 확인 성공

## 시나리오 2: 지뢰를 밟아 패배

**페르소나**: 빠른 플레이를 선호하는 경험자

**사전 조건**: 초급 게임 진행 중

### 흐름

1. 사용자가 타이머 `023` 상태에서 지뢰가 있는 셀을 좌클릭한다.
2. 즉시 패배 처리:
   - 클릭한 지뢰 셀이 빨간 배경(💣)으로 강조된다.
   - 모든 지뢰 셀이 공개된다.
   - 잘못 플래그한 셀(지뢰 없는 곳)에 ❌가 표시된다.
   - 스마일이 😵으로 변경된다.
   - 타이머가 `023`에서 정지된다.
3. 더 이상 셀 클릭이 동작하지 않는다.
4. 사용자가 😵 버튼을 클릭해 게임을 리셋한다.
   - 보드가 `idle` 상태로 초기화된다.
   - 타이머 `000`, 지뢰 카운터 `010`, 스마일 😀 복원.

**기대 결과**: 패배 시각화 완료 + 리셋 정상 동작

## 시나리오 3: 커스텀 난이도 설정

**페르소나**: 도전적인 설정을 즐기는 고급 사용자

**사전 조건**: 앱 접속

### 흐름

1. 사용자가 난이도 탭에서 **Custom**을 클릭한다.
2. 행/열/지뢰 입력 폼이 나타난다.
3. 사용자가 행=20, 열=20, 지뢰=80을 입력하고 **OK**를 클릭한다.
   - 입력값 검증: 20×20 = 400 칸, 최대 지뢰 = 400−9 = 391, 80 ≤ 391 ✓
   - 20×20 보드가 생성된다.
4. 플레이어가 게임을 완료한다.
5. 승리 후 점수 입력 모달이 **표시되지 않는다** (커스텀 난이도는 저장 불가).
6. 화면 상단에 "커스텀 기록은 저장되지 않습니다" 안내가 표시된다.

**기대 결과**: 커스텀 보드 정상 생성 + 점수 저장 비활성화 확인

## 시나리오 4: 키보드로만 게임 플레이

**페르소나**: 마우스 사용이 불편한 접근성 사용자

**사전 조건**: 앱 접속

### 흐름

1. 사용자가 `Tab` 키를 눌러 보드 첫 번째 셀에 포커스를 맞춘다.
2. 포커스된 셀에 파란색 포커스 링이 표시된다.
3. 방향키(↑↓←→)로 원하는 셀로 이동한다.
4. `Space`를 눌러 포커스된 셀을 공개한다.
5. `F`를 눌러 의심되는 셀에 플래그를 세운다.
6. `Tab`으로 스마일 버튼에 포커스 이동 후 `Enter`로 게임 리셋.
7. 모달이 열린 상태에서 `Escape`로 모달을 닫는다.

**기대 결과**: 마우스 없이 전체 게임 플레이 가능

---

## 시나리오 5: 리더보드 조회

**페르소나**: 자신의 기록 순위를 확인하려는 사용자

**사전 조건**: 이미 몇 개의 점수가 DB에 저장된 상태

### 흐름

1. 사용자가 우상단 🏆 트로피 버튼을 클릭한다.
2. 리더보드 모달이 열린다.
   - 현재 난이도(초급) 탭이 기본 선택된다.
   - `GET /api/scores?difficulty=beginner&limit=10` 요청
3. 상위 10개 기록이 순위/이름/시간/날짜 형식으로 표시된다.

| 순위 | 이름 | 시간 | 날짜 |
|---|---|---|---|
| 1 | Alice | 87초 | 2026-04-29 |
| 2 | Bob | 102초 | 2026-04-28 |
| 3 | Carol | 145초 | 2026-04-27 |

4. 사용자가 **Intermediate** 탭을 클릭한다.
   - `GET /api/scores?difficulty=intermediate&limit=10` 재조회
5. 사용자가 `Escape` 또는 ✕ 버튼으로 모달을 닫는다.

**기대 결과**: 난이도별 리더보드 정상 조회

## 시나리오 6: 모바일에서 플레이

**페르소나**: 스마트폰으로 접속한 사용자 (375px 너비)

**사전 조건**: 모바일 브라우저에서 앱 접속

### 흐름

1. 앱이 모바일 뷰포트에 맞게 레이아웃된다.
   - 셀 크기가 `clamp(24px, 4vw, 36px)` 공식으로 축소된다.
   - 보드가 가로로 스크롤 가능한 컨테이너 안에 배치된다.
2. 손가락으로 탭(좌클릭), 길게 누르기(우클릭 대체)로 조작한다.
3. 터치 영역이 최소 24px이므로 오조작이 최소화된다.
4. 난이도 탭이 작은 화면에서도 스크롤 없이 모두 표시된다.

**기대 결과**: 모바일에서 게임 플레이 가능 (가로 스크롤 허용)

## 시나리오 7: Chord Reveal (코드 공개)

**페르소나**: 고급 기법을 사용하는 숙련자

**사전 조건**: 숫자 2가 공개된 셀 주변에 정확히 2개의 플래그가 세워진 상태

### 흐름

1. 공개된 숫자 셀(값=2)에 중간 버튼을 클릭한다.
2. 인접 8칸 중 플래그 수(2) == 셀 값(2) 이므로 조건 충족.
3. 비플래그 인접 셀들이 자동으로 공개된다.
4. 만약 플래그가 잘못 세워진 경우(실제 지뢰 위치 오판), 잘못된 셀이 열리며 지뢰를 밟아 패배한다.

**기대 결과**: Chord Reveal 정상 동작 + 위험성 인지

## 에러 시나리오

### E-1: 백엔드 서버 미연결

1. 백엔드 서버(`localhost:3001`)가 꺼진 상태에서 앱 접속.
2. 게임 자체는 정상 동작 (프론트엔드 전용 기능).
3. 승리 후 점수 제출 시 "서버 연결 실패" 에러 메시지 표시.
4. 리더보드 모달 열면 "데이터를 불러올 수 없습니다" 표시.

### E-2: 잘못된 커스텀 입력

1. 지뢰 수에 `0`을 입력하고 OK 클릭.
2. "지뢰는 최소 1개 이상이어야 합니다" 에러 메시지.
3. 보드가 재생성되지 않음.
