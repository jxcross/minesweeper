# 제품 요구사항 문서 (PRD)

## 1. 제품 개요

### 1.1 비전

클래식 Windows Minesweeper의 게임플레이를 브라우저에서 완벽히 재현하되, 반응형 디자인·키보드 접근성·온라인 리더보드를 추가한 현대적 웹 게임.

### 1.2 목표

- 사용자가 별도 설치 없이 브라우저에서 즉시 플레이할 수 있다.
- 초급~고급 난이도와 커스텀 보드를 지원한다.
- 전 세계 플레이어와 기록을 비교할 수 있는 리더보드를 제공한다.

---

## 2. 기능 요구사항

### FR-1: 보드 렌더링

- **FR-1.1** 보드는 CSS Grid로 렌더링되며, 각 셀은 현재 상태(hidden/revealed/flagged/questioned)에 따라 다른 시각적 표현을 가진다.
- **FR-1.2** 공개된 셀은 숫자(1~8) 또는 빈 칸(0)을 표시한다. 숫자는 표준 색상(1=파랑, 2=초록, 3=빨강 등)을 따른다.
- **FR-1.3** 패배 시 모든 지뢰가 공개되고, 클릭한 지뢰 칸은 빨간 배경으로 강조된다.
- **FR-1.4** 잘못 플래그한 칸(지뢰가 없는데 플래그)은 패배 시 ❌로 표시된다.

### FR-2: 게임 컨트롤

- **FR-2.1** 좌클릭: `hidden` 또는 `questioned` 상태 셀 공개
- **FR-2.2** 우클릭: `hidden → flagged → questioned → hidden` 상태 순환 (컨텍스트 메뉴 방지)
- **FR-2.3** 중간 버튼 클릭 또는 더블클릭: 공개된 숫자 셀에서 Chord Reveal 실행
- **FR-2.4** 스마일 버튼: 현재 난이도로 게임 리셋
- **FR-2.5** 난이도 선택기: 탭 형태로 초급/중급/고급/커스텀 전환

### FR-3: 첫 클릭 안전

- **FR-3.1** 게임 시작 후 첫 번째 좌클릭 시점에 지뢰를 배치한다.
- **FR-3.2** 첫 클릭 셀과 인접 8칸(3×3 영역)에는 지뢰를 배치하지 않는다.
- **FR-3.3** 보드가 너무 작아 안전지대를 완전히 확보할 수 없는 경우, 클릭 셀만 제외한다.

### FR-4: 타이머

- **FR-4.1** 첫 번째 셀 클릭 시 타이머가 시작된다.
- **FR-4.2** 타이머는 1초 단위로 증가하며 3자리 디스플레이(000~999)로 표시된다.
- **FR-4.3** 승리 또는 패배 시 타이머가 정지된다.
- **FR-4.4** 게임 리셋 시 타이머가 000으로 초기화된다.

### FR-5: 지뢰 카운터

- **FR-5.1** `지뢰 수 - 플래그 수` 값을 3자리 디스플레이로 표시한다.
- **FR-5.2** 음수도 표시한다 (예: -5 → `-05`).
- **FR-5.3** 플래그 상태 변경 시 실시간으로 업데이트된다.

### FR-6: 승리/패배 처리

- **FR-6.1** 승리:
  - 모든 비지뢰 셀이 공개되면 승리
  - 미플래그 지뢰에 자동으로 🚩 표시
  - 스마일 버튼이 😎으로 변경
  - Confetti 애니메이션 재생 (3초)
  - 타이머 정지
- **FR-6.2** 패배:
  - 지뢰 셀 클릭 시 패배
  - 모든 지뢰 공개 (클릭 지뢰는 빨간 배경)
  - 잘못된 플래그에 ❌ 표시
  - 스마일 버튼이 😵으로 변경
  - 타이머 정지

### FR-7: 리더보드 (High Scores)

- **FR-7.1** 초급/중급/고급 난이도에서 승리 시 점수 입력 다이얼로그를 표시한다.
- **FR-7.2** 플레이어 이름(1~30자)을 입력하고 Submit하면 `POST /api/scores`로 기록을 저장한다.
- **FR-7.3** Skip 선택 시 점수를 저장하지 않고 진행한다.
- **FR-7.4** 트로피 아이콘 버튼으로 리더보드 모달을 열 수 있다.
- **FR-7.5** 리더보드는 난이도별 탭으로 구분되며 상위 10개 기록을 표시한다.
- **FR-7.6** 커스텀 난이도 기록은 저장하지 않는다.

### FR-8: 키보드 접근성

- **FR-8.1** 방향키로 보드 내 셀 포커스 이동
- **FR-8.2** `Space` / `Enter`: 포커스된 셀 공개
- **FR-8.3** `F`: 포커스된 셀 플래그/언플래그
- **FR-8.4** `Tab`: 스마일 버튼 / 난이도 선택기로 포커스 이동
- **FR-8.5** `Escape`: 열린 모달 닫기
- **FR-8.6** 포커스된 셀에 시각적 포커스 링 표시

### FR-9: 반응형 디자인

- **FR-9.1** 뷰포트 너비에 따라 셀 크기가 자동 조정된다 (`clamp(24px, 4vw, 36px)`).
- **FR-9.2** 모바일(< 640px)에서 보드가 가로로 스크롤 가능하다.
- **FR-9.3** 최소 셀 크기 24px로 터치 영역을 확보한다.

### FR-10: 커스텀 난이도

- **FR-10.1** 행(5~30), 열(5~50), 지뢰(1 ~ rows×cols−9) 범위에서 사용자 정의 보드를 생성한다.
- **FR-10.2** 입력값이 범위를 벗어나면 경계값으로 자동 조정되거나 에러 메시지를 표시한다.

## 3. 비기능 요구사항

| ID | 요구사항 | 기준 |
|---|---|---|
| NFR-1 | 초기 로딩 속도 | 4G 환경에서 FCP < 1.5초 |
| NFR-2 | 게임 로직 독립성 | `src/logic/` 함수는 React 의존성 없음 (순수 함수) |
| NFR-3 | API 응답 속도 | 리더보드 GET < 100ms (SQLite 인덱스 활용) |
| NFR-4 | 브라우저 호환성 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| NFR-5 | 접근성 | WCAG 2.1 AA 준수 (aria-label, role, focus visible) |
| NFR-6 | 테스트 커버리지 | 게임 로직 90%+, 컴포넌트 70%+ |

---

## 4. 기술 제약

- 백엔드 없이 게임 자체는 동작해야 한다 (API 실패 시 리더보드만 비활성화).
- `better-sqlite3`는 동기 API를 사용하므로 단일 스레드 Express에서 블로킹 I/O 최소화.
- DB 파일 경로: `backend/data/minesweeper.db`

## 5. 화면 목록

| 화면 | 설명 |
|---|---|
| 메인 게임 화면 | 보드 + 헤더(타이머/지뢰카운터/스마일) + 난이도 선택기 |
| 점수 입력 모달 | 승리 시 이름 입력 후 점수 제출 |
| 리더보드 모달 | 난이도별 탭 + 상위 10개 기록 테이블 |
| 커스텀 난이도 폼 | 커스텀 탭 선택 시 행/열/지뢰 입력 폼 |

---

## 6. 성공 지표

- 게임 완료율 (승리 + 패배) / 전체 세션 ≥ 80%
- 리더보드 점수 제출률 (승리 후 이름 입력) ≥ 50%
- 페이지 이탈률 < 20% (3초 이내)
