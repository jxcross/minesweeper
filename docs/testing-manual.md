# 수동 테스트 매뉴얼 (Manual Testing Guide)

## 준비사항

```bash
# 백엔드 시작 (터미널 1)
cd backend
npm install
npm run dev   # http://localhost:3001

# 프론트엔드 시작 (터미널 2)
cd frontend
npm install
npm run dev   # http://localhost:5173

# 브라우저에서 http://localhost:5173 접속
```

---

## 자동화 테스트 실행

```bash
# 백엔드 테스트 (15개)
cd backend && npm test

# 프론트엔드 테스트 (27개)
cd frontend && npm test

# 빌드 확인
cd frontend && npm run build
```

## 수동 테스트 시나리오

### M-01: 기본 게임 시작

| 단계 | 동작 | 기대 결과 |
|---|---|---|
| 1 | 브라우저에서 http://localhost:5173 접속 | 초급(9×9) 보드 표시, 지뢰 카운터 010, 타이머 000, 😀 |
| 2 | 보드 임의 셀 클릭 | 타이머 시작, 셀 및 주변 공개 |
| 3 | 스마일 버튼(😀) 클릭 | 보드 초기화, 타이머 000 |

**통과 기준**: 초기 상태 정상 표시, 첫 클릭 후 타이머 시작

---

### M-02: 첫 클릭 안전 검증

| 단계 | 동작 | 기대 결과 |
|---|---|---|
| 1 | 초급 보드에서 (0,0) 셀 클릭 | 패배하지 않음 |
| 2 | (0,0) 및 주변 3×3 범위 확인 | 어떤 셀도 지뢰 없음 |
| 3 | 같은 과정을 5회 반복 | 항상 안전 |

**통과 기준**: 첫 클릭 위치와 인접 8칸에 지뢰 없음 (10회 반복 확인)

## API 수동 테스트 (curl)

```bash
# 점수 등록
curl -X POST http://localhost:3001/api/scores \
  -H "Content-Type: application/json" \
  -d '{"playerName":"TestUser","difficulty":"beginner","timeSeconds":87}'

# 리더보드 조회
curl "http://localhost:3001/api/scores?difficulty=beginner&limit=5"

# 유효성 검사 오류 확인
curl -X POST http://localhost:3001/api/scores \
  -H "Content-Type: application/json" \
  -d '{"playerName":"Test","difficulty":"custom","timeSeconds":50}'
# → 400 Validation failed

# 헬스 체크
curl http://localhost:3001/health
# → {"status":"ok"}
```
